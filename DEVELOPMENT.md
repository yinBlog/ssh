# SSH Studio 开发指南\n\n## 环境要求\n\n- **Node.js**: 18.0.0 或更高版本\n- **npm**: 8.0.0 或更高版本\n- **Git**: 用于版本控制\n\n## 快速开始\n\n### 1. 克隆项目\n\n```bash\ngit clone https://github.com/yinBlog/ssh.git\ncd ssh\n```\n\n### 2. 安装依赖\n\n```bash\nnpm install\n```\n\n首次安装会下载 Electron，可能需要 5-10 分钟，取决于网络速度。\n\n### 3. 启动开发模式\n\n```bash\n# 普通启动，自动打开开发者工具\nnpm start\n\n# 或使用远程调试端口(便于外部工具连接)\nnpm run dev\n```\n\n## 构建与打包\n\n### 开发构建\n\n```bash\n# 检查构建配置(不打包)\nnpm run pack\n```\n\n### 生产构建\n\n```bash\n# 为当前平台打包\nnpm run build\n\n# 仅 Windows\nnpm run build:win\n\n# 仅 macOS\nnpm run build:mac\n\n# 仅 Linux\nnpm run build:linux\n\n# 所有平台\nnpm run build:all\n```\n\n输出文件在 `dist/` 目录下。\n\n## 项目结构\n\n```\nssh/\n├── src/\n│   ├── main.js                # 主进程入口\n│   ├── preload.js             # 渲染进程安全桥接\n│   ├── lib/\n│   │   ├── logger.js          # 结构化日志模块\n│   │   ├── error-handler.js   # 统一错误处理\n│   │   └── validators.js      # 输入验证\n│   └── renderer/\n│       ├── index.html         # UI 结构\n│       ├── styles.css         # 样式\n│       └── renderer.js        # UI 逻辑\n├── assets/                    # 应用图标、资源\n├── .github/\n│   └── workflows/             # GitHub Actions CI/CD\n├── package.json               # 项目配置 & 构建脚本\n├── DEVELOPMENT.md             # 本文件\n└── README.md\n```\n\n## 代码规范\n\n### JavaScript\n\n- 使用 `'use strict'` 在文件顶部\n- 使用 `const` 而非 `var`\n- 使用箭头函数处理异步\n- 异常必须捕获，避免未处理的 Promise rejection\n- 缩进使用 2 个空格\n\n### 命名约定\n\n- **函数名**: camelCase (e.g., `createWindow`, `connectToHost`)\n- **常量名**: UPPER_SNAKE_CASE (e.g., `STAT_CMD`, `DEFAULT_TIMEOUT`)\n- **类名**: PascalCase (e.g., `SSHConnection`, `Logger`)\n- **私有方法**: 前缀下划线 (e.g., `_parseError`)\n\n### 注释规范\n\n```javascript\n// 简单注释用双斜杠\n\n/**\n * 函数说明文档\n * @param {Type} name - 参数描述\n * @returns {Type} 返回值描述\n */\nfunction example(name) {\n  // 实现\n}\n```\n\n## IPC 通信\n\n主进程与渲染进程通过 `window.sshBridge` 通信(定义在 `src/preload.js`)。\n\n### 添加新的 IPC 接口步骤\n\n#### 1. 主进程 (`src/main.js`)\n\n```javascript\nipcMain.handle('feature:action', async (_e, arg) => {\n  // 处理逻辑\n  try {\n    const result = await doSomething(arg);\n    return { ok: true, data: result };\n  } catch (err) {\n    return { ok: false, error: err.message };\n  }\n});\n```\n\n#### 2. 预加载脚本 (`src/preload.js`)\n\n```javascript\ncontextBridge.exposeInMainWorld('sshBridge', {\n  // 现有接口...\n  featureAction: (arg) => ipcRenderer.invoke('feature:action', arg),\n});\n```\n\n#### 3. 渲染进程 (`src/renderer/renderer.js`)\n\n```javascript\nconst result = await window.sshBridge.featureAction(arg);\nif (result.ok) {\n  console.log('Success:', result.data);\n} else {\n  console.error('Error:', result.error);\n}\n```\n\n## 模块说明\n\n### `src/lib/logger.js` - 结构化日志\n\n```javascript\nconst logger = require('./lib/logger');\n\nlogger.log('Connection established', { host: '192.168.1.1' });\nlogger.warn('High memory usage', { usage: '85%' });\nlogger.error('Connection failed', error);\nlogger.debug('Debug info', { details: '...' });\n```\n\n日志文件保存在用户数据目录的 `logs/` 文件夹，每天一个文件。\n\n### `src/lib/error-handler.js` - 错误处理\n\n```javascript\nconst { parseSSHError, getUserFriendlyMessage } = require('./lib/error-handler');\n\ntry {\n  await conn.connect(config);\n} catch (err) {\n  const sshError = parseSSHError(err);\n  const userMsg = getUserFriendlyMessage(sshError);\n  console.error(userMsg);\n}\n```\n\n### `src/lib/validators.js` - 输入验证\n\n```javascript\nconst { validateHostConfig } = require('./lib/validators');\n\nconst result = validateHostConfig(hostConfig);\nif (!result.valid) {\n  result.errors.forEach(err => console.error(err));\n}\n```\n\n## 调试技巧\n\n### 开发者工具\n\n- 启动时自动打开(参见 `src/main.js` 第 172 行)\n- 按 **F12** 随时切换\n- 在非终端区域按 F12 才能生效(因为终端会捕获按键)\n- 使用 `console.log` 输出到控制台\n\n### 远程调试\n\n```bash\nnpm run dev  # 启动 --remote-debugging-port=9223\n# 在 Chrome 中访问 chrome://inspect\n```\n\n### 查看日志\n\n```bash\n# Windows\n%APPDATA%\\ssh-connect-tool\\logs\n\n# macOS\n~/Library/Application Support/ssh-connect-tool/logs\n\n# Linux\n~/.local/share/ssh-connect-tool/logs\n```\n\n## 测试\n\n当前项目暂无自动化测试。建议后续集成：\n\n```bash\n# 单元测试（待集成）\nnpm test\n\n# E2E 测试（待集成）\nnpm run test:e2e\n```\n\n## 常见问题\n\n### Q: 编译失败 \"Cannot find module 'ssh2'\"\n\nA: 运行 `npm rebuild` 重建原生模块。\n\n```bash\nnpm rebuild\n```\n\n### Q: Windows 打包失败\n\nA: 确保已安装 Visual Studio Build Tools 或类似工具，以支持原生模块编译。\n\n或在 npm 中安装全局工具：\n\n```bash\nnpm install --global windows-build-tools\n```\n\n### Q: 如何修改应用图标\n\nA: 将 PNG/ICO 图片放在 `assets/` 目录，在 `package.json` 的 `build` 配置中指定路径。\n\n### Q: 如何修改应用名称\n\nA: 在 `package.json` 中修改 `name` 字段和 `build.productName`。\n\n### Q: 如何查看所有可用命令\n\nA: 运行 `npm run` 查看所有脚本。\n\n## 贡献指南\n\n1.
